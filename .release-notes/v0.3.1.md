# v0.3.1 — Touch-Friendly UI + Flexible Reg# + SVK Cache

**Release date:** 2026-06-26

## Highlights

### Touch-Friendly Segmented Controls

Replaced native `<select>` dropdowns with **button-row segmented
controls** for short option lists. Inspired by the reference image
you shared — each option is a big, rounded, touch-friendly button
with a clearly visible selected state.

| Where | Options |
|-------|---------|
| Starting score (x01) | 121 / 170 / 301 / 501 / 701 / 901 |
| Elimination Format | Single elim / Double elim / RR / Double RR / RR → KO |
| Game mode | x01 / Cricket / Shanghai |
| Legs to win | 1 / 2 / 3 / 5 / 7 |
| Max darts per leg | 0 / 20 / 30 / 45 / 50 / 100 |
| Seeding | Ordered / Random |
| Number of groups | 1 / 2 / 4 |
| Players advancing per group | 1 / 2 / 3 / 4 |
| Number of teams | 2 / 3 / 4 |
| Legs per set | 1 (single) / First to 2-7 |
| Sets per match | 1 (single) / Best of 2-7 |
| Shanghai rounds | 5 / 7 / 9 / 12 / 20 |

**Long lists preserved** as scrollable grids (player picker) or
dropdowns (club filter, format preset). Segmented control is for
short lists only — too many buttons would wrap and become
unreadable.

### x01 Starting Scores: 121 and 170 Added

Two new starting scores per your request:
- **121** — beginner / quick matches
- **170** — high-skill bracket (170 is the highest checkout in darts)

Available scores: `121 / 170 / 301 / 501 / 701 / 901`.

### Registration Number Format: Any Separator or None

`regNumber` now accepts any non-alphanumeric character as a
separator, including no separator at all:

| Input | Previously | Now |
|-------|------------|-----|
| `NR#100298` | ✓ | ✓ |
| `NR.100298` | ✗ | ✓ |
| `NR-100298` | ✗ | ✓ |
| `NR?100298` | ✗ | ✓ |
| `NR100298` (no separator) | ✗ | ✓ |
| `SVK003112` | ✓ | ✓ |

New pattern: `/^[A-Z]+[^A-Z0-9]?\d{6}$/`.

### Local SVK License Cache + Online Lookup via CORS Proxies

Two complementary paths for finding players by SVK federation
license:

1. **Online (CORS proxy chain).** `lookupPlayerOnSVK()` tries
   corsproxy.io → api.allorigins.win → cors-anywhere.herokuapp.com.
   If a proxy works, the SVK portal HTML is parsed and rows
   returned. The browser DevTools still shows a CORS warning —
   that's the browser's informational note, not a real failure.
2. **Local cache.** `Settings → Import SVK license list` lets the
   admin paste the full SVK portal table once. Stored in a new
   `svk_players` IndexedDB store. Lookups work offline.

`DB_VERSION` bumped 3 → 4 to add the new store.

### New-Competition Form: Three Orthogonal Dimensions

After much back-and-forth we settled on:

| Field | Values | Where in form |
|-------|--------|---------------|
| **Name** | text | top |
| **Elimination Format** | Single elim, Double elim, RR, Double RR, RR → KO | second |
| **Season / Round / Notes** | metadata | third |
| **Type** | Tournament / League | below notes |
| **Participant Format** | Singles / Doubles / Teams | below notes |

Type + Participant Format were moved below Notes per your
feedback. Each section is a row of touch-friendly buttons with the
selected one highlighted in green.

### Player Picker Refactor

- **Scrollable grid** (`max-height: 360px; overflow-y: auto`)
- **Substring search** (case-insensitive across surname + firstName
  + middle + nameSuffixes). Typing `eme` matches all surnames
  containing `eme`.
- **Participants grid** below the picker. Each picked player shows
  as a card with name + metadata, and a red ✕ button to remove.

### DB Field: Name Suffixes

Added `nameSuffixes` field to the `players` schema. For names like
`John Smith Jr.`, the parser stores middle="", nameSuffixes="Jr.".
`formatPlayerName` now displays "Surname, FirstName [Middle]
[Suffixes]" — e.g. "Slovák, Ján ml. Jr.".

The "+ New player" modal also has a "Name suffixes" input field.

### Game Mode Rename: `'01'` → `'x01'`

The dart community uses "x01" as the umbrella term for 301 / 501 /
etc. Updated across the project: engine, stats, screens, store,
tests, competition.js. Tile label changed from "01 Games" to "x01
Games".

### Dropdown Readability Fix

Native `<select>` elements had unreadable text on the dark theme.
Added `.dropdown-readable` CSS rule with explicit dark background
(`#1a212c`) and light text (`#e6ecf5`). Selected option inside the
dropdown also uses the same colors.
